Mins Bot

A desktop AI agent that runs locally on your machine. It browses the web, handles your files, drives your messaging tools, and runs an installable library of skills — without sending your data anywhere.

stable v0.4.2 macOS · Windows · Linux ≈ 42 MB
TL;DR. One binary. No account. No server. Model calls go direct from your device to the provider you pick.

# Installation

Pick the package for your platform and run the installer. The binary is signed and self-contained.

# macOS — via Homebrew $ brew install mins-bot # Linux — via the AppImage (or distro package) $ chmod +x mins-bot-0.4.2.AppImage $ ./mins-bot-0.4.2.AppImage # Windows — double-click the .msi

# First run

On first launch, Mins Bot creates a config directory at ~/.mins and prompts you to pick a model provider. No account is created — credentials land in the OS keychain.

$ mins init → Creating config at ~/.mins ... → Select provider: (anthropic, openai, local) > anthropic → API key: ******** ✓ Saved to OS keychain.

# Capabilities

The agent exposes four primitives. They compose.

01 · Autonomous browsing

Navigate, read, and act on the real web under your own session. Mins Bot drives Chromium in a per-task sandbox.

Note. Every browse session runs under your browser profile. No traffic is routed through Mins infrastructure.

02 · Filesystem access

The agent reads and writes only within paths you've explicitly allow-listed. Every change is journaled to a local log you can inspect or revert.

03 · Messaging integrations

Slack, Discord, Gmail, and more. Tokens rest in the OS keychain — we never see them. Revoke access at the provider's security page at any time.

04 · Skills inventory

Skills are small manifest-plus-code packages the agent can discover and compose. Bring your own, pin to a shared registry, or write one in an afternoon.

# Data policy

In one line. Your data never leaves your device. We do not operate a server. Nothing of yours enters any training pipeline.
